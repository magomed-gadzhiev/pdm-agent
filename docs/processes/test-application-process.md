# Тестовый бизнес-процесс "Обработка заявки"

## Описание

Тестовый бизнес-процесс для демонстрации работы системы управления бизнес-процессами. Процесс состоит из трех этапов обработки заявки: создание, рассмотрение и утверждение.

## Структура процесса

### Этап 1: Создание заявки
- **Задача:** Создание заявки
- **Документ для редактирования:** Заявка
- **Описание:** Первый этап процесса - создание заявки с указанием основных данных

### Этап 2: Рассмотрение заявки
- **Задача:** Рассмотрение заявки
- **Документы для чтения:** Заявка
- **Документ для редактирования:** Решение по заявке
- **Описание:** Второй этап - рассмотрение созданной заявки и принятие решения

### Этап 3: Утверждение заявки
- **Задача:** Утверждение заявки
- **Документы для чтения:** Заявка, Решение по заявке
- **Документ для редактирования:** Утверждение заявки
- **Описание:** Финальный этап - утверждение заявки на основе решения

## Созданные сущности

### Справочники

#### Статусы заявок (ID: 1)
- **Поля:**
  - `название` (строка, обязательное, уникальное) - название статуса
  - `код` (строка, обязательное, уникальное) - код статуса

### Типы документов

#### Заявка (ID: 1)
- **Поля:**
  - `номер` (строка, обязательное, уникальное) - номер заявки
  - `дата` (дата, обязательное) - дата создания заявки
  - `название` (строка, обязательное) - название заявки
  - `описание` (текст, необязательное) - описание заявки

#### Решение по заявке (ID: 2)
- **Поля:**
  - `номер` (строка, обязательное, уникальное) - номер решения
  - `дата` (дата, обязательное) - дата решения
  - `решение` (текст, обязательное) - текст решения
  - `комментарий` (текст, необязательное) - комментарий к решению

#### Утверждение заявки (ID: 3)
- **Поля:**
  - `номер` (строка, обязательное, уникальное) - номер утверждения
  - `дата` (дата, обязательное) - дата утверждения
  - `утверждено` (булево, обязательное) - статус утверждения
  - `комментарий` (текст, необязательное) - комментарий к утверждению

### Бизнес-процесс

#### Обработка заявки (ID: 1)
- **Описание:** Тестовый бизнес-процесс обработки заявки с тремя этапами
- **Задачи:**
  1. Создание заявки (ID: 1, порядок: 1)
  2. Рассмотрение заявки (ID: 2, порядок: 2)
  3. Утверждение заявки (ID: 3, порядок: 3)

## Статус настройки

### ✅ Полностью настроено

Все компоненты бизнес-процесса успешно созданы и настроены:

1. **Справочники** - создан справочник "Статусы заявок"
2. **Типы документов** - созданы 3 типа документов с полями
3. **Бизнес-процесс** - создан и привязаны типы документов
4. **Задачи** - созданы 3 задачи с правильной последовательностью
5. **Условия запуска** - настроены автоматически:
   - Задача "Рассмотрение заявки" запускается после завершения "Создание заявки"
   - Задача "Утверждение заявки" запускается после завершения "Рассмотрение заявки"
6. **Привязка документов** - все документы привязаны к задачам:
   - **Задача "Создание заявки"**: редактирование - Заявка (ID: 1)
   - **Задача "Рассмотрение заявки"**: чтение - Заявка (ID: 1), редактирование - Решение по заявке (ID: 2)
   - **Задача "Утверждение заявки"**: чтение - Заявка (ID: 1), Решение по заявке (ID: 2), редактирование - Утверждение заявки (ID: 3)

**Бизнес-процесс готов к использованию!** ✓

## Примечания

Все настройки выполнены автоматически через MCP API. Скрипт `scripts/setup_test_process.py` больше не требуется, но сохранен для справки.

## Использование

После завершения настройки процесс готов к использованию:

1. Создайте экземпляр процесса через веб-интерфейс или API
2. Запустите экземпляр процесса
3. Выполните задачи по порядку:
   - Создание заявки
   - Рассмотрение заявки
   - Утверждение заявки
